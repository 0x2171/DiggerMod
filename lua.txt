# Lua API для модов DiggerSteam

Документ описывает Lua API, доступный мод-скриптам в игре.
Фокус документа: практическое использование в модах.

## 1. Быстрый старт

### Что доступно в обычном Lua-скрипте объекта
Если скрипт подключен через `LuaBehaviour`, внутри Lua уже есть:

- `self` — объект `LuaBehaviour`
- `transform` — трансформ объекта
- `gameObject` — текущий GameObject
- `Game`
- `Chat`
- `Player`
- `World`
- `Physics`
- `Utils`
- `Skybox`
- дополнительные переменные из инъекций (`LuaBehaviourTag.Injections`)

### Правило вызова

- Методы: `Object:Method(...)`
- Свойства: `Object.Property`
- Для Unity/C# типов используйте `CS.*`, например:
  - `CS.UnityEngine.Vector3(1,2,3)`
  - `CS.BlockType.WoodPlank`

### Минимальный шаблон

```lua
local Unity = CS.UnityEngine

function Start()
    Chat:AddMessage("Lua script started", "[00FF00]")
end

function Update()
end

function OnDestroy()
end
```

## 2. Callback-функции LuaBehaviour

Эти функции (если объявлены в Lua) вызываются движком автоматически:

| Функция | Когда вызывается | Параметры |
|---|---|---|
| `awake()` | после привязки Lua-таблицы | нет |
| `Start()` | старт объекта | нет |
| `Update()` | каждый кадр | нет |
| `OnDestroy()` | уничтожение объекта | нет |
| `OnInteract()` | взаимодействие игрока с объектом | нет |
| `OnInteractChild(go)` | взаимодействие с дочерним объектом | `go: GameObject` |
| `OnLook()` | игрок навелся на объект | нет |
| `OnHit()` | попадание/удар по объекту | нет |
| `OnPlayerCollision()` | коллизия с игроком | нет |
| `OnTriggerEnter(collider)` | вход в trigger | `collider: Collider` |
| `ReceiveEvent(eventName, arg)` | получение события от `self:SendEvent` | `eventName: string`, `arg: object[]` |
| `OnGUI()` | GUI-событие Unity | нет |
| `OnRenderObject()` | рендер-событие Unity | нет |

Важно для `ReceiveEvent`: `arg` — C#-массив, индексация обычно с `0` (`arg[0]`, `arg[1]`, ...).

## 3. API объекта `self` (`LuaBehaviour`)

### Методы

### `self:IsOwner()`
- Что делает: проверяет, является ли локальный клиент владельцем объекта.
- Возвращает: `bool`.

```lua
if self:IsOwner() then
    Chat:AddMessage("Я владелец объекта")
end
```

### `self:SendEvent(eventName, cached, ...)`
- Что делает: отправляет Lua-событие другим клиентам/экземплярам скрипта.
- Параметры:
  - `eventName: string`
  - `cached: bool`
  - `...` любые дополнительные аргументы
- Возвращает: `nil`.

```lua
self:SendEvent("SwitchDoor", true, true) -- пример: кэшируем состояние "открыто"
```

### `self:GetPhotonView()`
- Что делает: возвращает PhotonView объекта.
- Возвращает: `PhotonView`.

```lua
local view = self:GetPhotonView()
if view ~= nil then
    Chat:AddMessage("ViewID: " .. tostring(view.viewID))
end
```

### `self:AttachLuaScript([luaTable])`
- Что делает: вручную перепривязывает Lua-скрипт.
- Возвращает: `nil`.
- Примечание: служебный метод, в обычных модах почти не нужен.

### `self:DisposeLuaReference()`
- Что делает: освобождает Lua-ссылки объекта.
- Возвращает: `nil`.
- Примечание: служебный метод.

### Прочие доступные методы
`self:OnInteract()`, `self:OnInteractChild(go)`, `self:OnRaycastStart()`, `self:OnHit()`, `self:Start()`

Это публичные методы компонента. Обычно их не вызывают вручную из скрипта, а используют callback-функции раздела 2.

### Свойства

| Свойство | Тип | Доступ | Описание |
|---|---|---|---|
| `self.ItemDefenitionName` | `string` | чтение | имя ItemDefinition (если задан) |
| `self.luaScript` | `TextAsset` | чтение/запись | исходный Lua-ассет |
| `self.injections` | `Inject[]` | чтение/запись | инъекции |
| `self.LuaScript` | `LuaTable` | чтение/запись | текущая Lua-таблица скрипта |
| `self.Item` | `ItemDefinition` | чтение/запись | связанный item |

## 5. `Chat`

### Методы

### `Chat:Start()`
- Что делает: включает подписку на ошибки Lua в чат.
- Возвращает: `nil`.

```lua
Chat:Start()
```

### `Chat:AddMessage(text [, color])`
- Что делает: добавляет локальное сообщение в чат.
- Параметры:
  - `text: string`
  - `color: string` (по умолчанию `"[FFFFFF]"`)
- Возвращает: `nil`.

```lua
Chat:AddMessage("Привет, мир", "[00FF00]")
```

### `Chat:SendMessage(text [, color])`
- Что делает: отправляет сообщение в игровой чат.
- Параметры:
  - `text: string`
  - `color: string` (по умолчанию `"[FFFFFF]"`)
- Возвращает: `nil`.

```lua
Chat:SendMessage("Я отправил сообщение в чат", "[FFD700]")
```

### `Chat:OnMessage(callback)`
- Что делает: подписывает callback на ввод текста.
- Параметры:
  - `callback: function(text)`
- Возвращает: `nil`.

```lua
Chat:OnMessage(function(text)
    Chat:AddMessage("Вы ввели: " .. text, "[AAAAFF]")
end)
```

### Свойства

| Свойство | Тип | Доступ | Описание | Пример |
|---|---|---|---|---|
| `Chat.IsEnable` | `bool` | чтение | активен ли чат | `if Chat.IsEnable then ... end` |
| `Chat.LogErrors` | `bool` | чтение/запись | печатать ли Lua-ошибки в чат | `Chat.LogErrors = true` |

## 6. `Player`

### Методы

### `Player:TakeDamage(damage)`
- Что делает: наносит урон локальному игроку.
- Параметры: `damage: number`
- Возвращает: `nil`.

```lua
Player:TakeDamage(5)
```

### `Player:OnPlayerCreated(callback)`
- Что делает: подписывает callback на событие создания/старта игрока.
- Параметры: `callback: function()`
- Возвращает: `nil`.

```lua
Player:OnPlayerCreated(function()
    Chat:AddMessage("Игрок создан")
end)
```

### `Player:BuildAndHitSpeedUp(value)`
- Что делает: ускоряет/возвращает скорость строительства и удара.
- Параметры: `value: bool`
- Возвращает: `nil`.

```lua
Player:BuildAndHitSpeedUp(true)
```

### `Player:IsControlVehicle(behaviour)`
- Что делает: проверяет, управляет ли игрок транспортом, связанным с `LuaBehaviour`.
- Параметры: `behaviour: LuaBehaviour`
- Возвращает: `bool`.

```lua
if Player:IsControlVehicle(self) then
    Chat:AddMessage("Игрок управляет этим транспортом")
end
```

### `Player:EnableFly(value)`
- Что делает: включает/выключает полет.
- Параметры: `value: bool`
- Возвращает: `nil`.

```lua
Player:EnableFly(true)
```

### `Player:EnableSpeedUp(value)`
- Что делает: включает/выключает ускорение.
- Параметры: `value: bool`
- Возвращает: `nil`.

```lua
Player:EnableSpeedUp(true)
```

### `Player:TakeBan()`
- Что делает: локально инициирует бан игрока.
- Возвращает: `nil`.
- Осторожно: административно опасная операция.

### Свойства

| Свойство | Тип | Доступ | Описание |
|---|---|---|---|
| `Player.Name` | `string` | чтение | ник игрока |
| `Player.IsAdmin` | `bool` | чтение | права администратора |
| `Player.IsModerator` | `bool` | чтение | права модератора |
| `Player.IsBuilder` | `bool` | чтение | права строителя |
| `Player.GetCurrentVehicle` | `GameObject or nil` | чтение | текущий транспорт |
| `Player.IsInVehicle` | `bool` | чтение | игрок в транспорте |
| `Player.IsFlying` | `bool` | чтение | включен ли режим полета |
| `Player.IsSpeedUp` | `bool` | чтение | включено ли ускорение |
| `Player.Camera` | `Camera` | чтение | камера игрока |
| `Player.RaycastHit` | `RaycastHit` | чтение | текущее hit-состояние raycast |
| `Player.RaycastDistance` | `number` | чтение/запись | дистанция raycast |
| `Player.CurrentBlock` | `int` | чтение | выбранный тип блока |
| `Player.CurrentKind` | `int` | чтение | выбранный kind блока |
| `Player.Position` | `Vector3` | чтение/запись | позиция игрока |

Примеры:

```lua
Chat:AddMessage("Игрок: " .. Player.Name)

if Player.IsInVehicle then
    local vehicle = Player.GetCurrentVehicle
    if vehicle ~= nil then
        Chat:AddMessage("Транспорт: " .. vehicle.name)
    end
end

Player.RaycastDistance = 8.0
Player.Position = CS.UnityEngine.Vector3(10, 20, 30)
```

## 7. `World`

### Важное про координаты
Внутри API координаты блока обрабатываются как `X, Z, Y`.
Если передаете обычный Unity `Vector3(x, y, z)` напрямую, оси `Y/Z` для блочного мира фактически меняются местами.

### Методы

### `World:DestroySphereAreaAt(position, radius)`
- Что делает: удаляет сферическую область блоков.
- Параметры:
  - `position: Vector3`
  - `radius: int`
- Возвращает: `nil`.

```lua
World:DestroySphereAreaAt(CS.UnityEngine.Vector3(50, 20, 50), 4)
```

### `World:AddBlock(position [, type [, kind]])`
- Что делает: ставит блок.
- Параметры:
  - `position: Vector3`
  - `type: int` (по умолчанию `7`)
  - `kind: int` (по умолчанию `0`)
- Возвращает: `nil`.

```lua
local V3 = CS.UnityEngine.Vector3
World:AddBlock(V3(10, 20, 5), CS.BlockType.WoodPlank, 0)
```

### `World:RemoveBlock(position)`
- Что делает: удаляет блок.
- Параметры: `position: Vector3`
- Возвращает: `nil`.

```lua
World:RemoveBlock(CS.UnityEngine.Vector3(10, 20, 5))
```

### `World:GetBlockType(position)`
- Что делает: получает тип блока.
- Параметры: `position: Vector3`
- Возвращает: `int`.

```lua
local t = World:GetBlockType(CS.UnityEngine.Vector3(10, 20, 5))
Chat:AddMessage("Type: " .. tostring(t))
```

### `World:GetBlockKind(position)`
- Что делает: получает kind блока.
- Параметры: `position: Vector3`
- Возвращает: `int`.

```lua
local k = World:GetBlockKind(CS.UnityEngine.Vector3(10, 20, 5))
Chat:AddMessage("Kind: " .. tostring(k))
```

### Свойства

| Свойство | Тип | Доступ | Описание |
|---|---|---|---|
| `World.SizeX` | `int` | чтение | размер мира по X |
| `World.SizeZ` | `int` | чтение | размер мира по Z |
| `World.SizeY` | `int` | чтение | размер мира по Y |
| `World.IsInfinite` | `bool` | чтение | `true` для InfiniteV6, иначе `false` |

```lua
Chat:AddMessage(("World size: X=%d Z=%d Y=%d"):format(World.SizeX, World.SizeZ, World.SizeY))

if World.IsInfinite then
    Chat:AddMessage("InfiniteV6 map")
else
    Chat:AddMessage("Legacy finite map")
end
```

## 8. `Physics`

### `Physics:Ray(ray, length)`
- Что делает: делает Raycast.
- Параметры:
  - `ray: Ray`
  - `length: int`
- Возвращает: `hitOk: bool`, `hitInfo: RaycastHit`.

```lua
local cam = Player.Camera
local ray = CS.UnityEngine.Ray(cam.transform.position, cam.transform.forward)
local ok, hit = Physics:Ray(ray, 100)
if ok then
    Chat:AddMessage("Попадание в: " .. hit.collider.gameObject.name)
end
```

## 9. `Utils`

### `Utils:QuaternionLerp(target, a, b, t)`
- Что делает: плавно выставляет `target.rotation = Quaternion.Lerp(a,b,t)`.
- Параметры:
  - `target: Transform`
  - `a: Quaternion`
  - `b: Quaternion`
  - `t: number`
- Возвращает: `nil`.

```lua
local tr = gameObject.transform
local a = tr.rotation
local b = CS.UnityEngine.Quaternion.Euler(0, 180, 0)
Utils:QuaternionLerp(tr, a, b, 0.1)
```

### `Utils:AttachLuaFromTag(go)`
- Что делает: если у объекта есть `LuaBehaviourTag`, добавляет `LuaBehaviour` и подключает скрипт.
- Параметры: `go: GameObject`
- Возвращает: `nil`.

```lua
local target = CS.UnityEngine.GameObject.Find("MyTaggedObject")
if target ~= nil then
    Utils:AttachLuaFromTag(target)
end
```

### Методы

### `Skybox:CreateDynamicSkybox()`
- Что делает: создает динамический skybox, если его еще нет.
- Возвращает: `nil`.

```lua
Skybox:CreateDynamicSkybox()
```

### Свойства

| Свойство | Тип | Доступ | Описание | Пример |
|---|---|---|---|---|

| `Skybox.Hour` | `number` | чтение/запись | час суток | `Skybox.Hour = 14` |
| `Skybox.ProgressTime` | `bool` | чтение/запись | идет ли время | `Skybox.ProgressTime = false` |
| `Skybox.DayLengthInMinutes` | `number` | чтение/запись | длительность дня (мин) | `Skybox.DayLengthInMinutes = 20` |

## 11. Дополнительно доступно в Lua

### `Transform:FindDeepChild(name)`
Доступно у `Transform` как extension-метод.
Ищет дочерний объект по имени в ширину (BFS), рекурсивно.

```lua
local t = transform:FindDeepChild("DoorHandle")
if t ~= nil then
    Chat:AddMessage("Нашли: " .. t.name)
end
```

## 13. Частые паттерны

### 13.1 Событие с синхронизацией состояния

```lua
local isOpen = false

function OnInteract()
    isOpen = not isOpen
    self:SendEvent("SwitchState", true, isOpen)
end

function ReceiveEvent(eventName, arg)
    if eventName == "SwitchState" then
        local value = arg[0]
        Chat:AddMessage("Новое состояние: " .. tostring(value))
    end
end
```

### 13.2 Поставить блок перед игроком

```lua
function OnInteract()
    local cam = Player.Camera
    local pos = cam.transform.position + cam.transform.forward * 3
    World:AddBlock(pos, CS.BlockType.Brick, CS.BlockKind.Default)
end
```
